## Description

PI has zero shell completion support. Users typing `pi run <TAB>` should see available automations, `pi info <TAB>` should complete automation names, etc. Cobra has built-in support for generating shell completion scripts (bash, zsh, fish, powershell), and also supports custom `ValidArgsFunction` on individual commands for dynamic argument completion.

This task adds:
1. A `pi completion` command that outputs shell completion scripts (using Cobra's built-in generator)
2. Dynamic completion functions for commands that take automation names as arguments (`run`, `info`, `doctor` doesn't need one since it takes no args)
3. Integration with `pi shell` so that completion is automatically available when shortcuts are installed

## Implementation Notes

**Approach**: Cobra's completion system works as follows:
- `cobra.Command` has a `ValidArgsFunction` field that returns completions dynamically
- Cobra has built-in `GenBashCompletionV2()`, `GenZshCompletion()`, `GenFishCompletion()`, `GenPowerShellCompletion()` methods
- Standard pattern is a `completion` subcommand with shell name as argument

For dynamic completion of automation names:
- Use `resolveProject()` + `Discover()` to get the list of automations
- Return them as `cobra.ShellCompDirective` completions
- Handle errors silently (completion should never crash â€” just return empty)

For `pi shell` integration:
- The shell file generated by `pi shell` sources `~/.pi/shell/*.sh`
- We can generate a completion setup alongside the shortcut file
- Actually, the simplest approach: include completion setup in the `_pi-wrapper.sh` file that already exists
- Or: just document `pi completion zsh > ~/.pi/completion.zsh` and source it
- Decision: Keep it simple. The `pi completion` command outputs the script. Users can pipe it or add it to their shell config. `pi shell` can automatically set up completion.

**Key design decisions:**
- `pi completion` uses Cobra's built-in generation methods â€” no custom logic needed for the shell scripts themselves
- `ValidArgsFunction` on `run` and `info` uses `resolveProject` + `Discover` to list available automations
- Completion for `run` includes both workspace and package automations (not builtins by default, since they're hidden in `pi list` too)
- Errors in completion silently return empty list (never crash the shell)
- Added completion setup to `pi shell` â€” writes a completion script to `~/.pi/shell/_pi-completion.{zsh,bash}` and it's automatically sourced via the existing glob pattern
