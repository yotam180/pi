## Description
Create a Homebrew tap at `github.com/yotam180/homebrew-pi` and wire GoReleaser to auto-update the cask on every release. After this task, `brew install yotam180/pi/pi` works on macOS for both Intel and Apple Silicon.

- [x] Both `darwin/amd64` and `darwin/arm64` are covered (Homebrew handles the arch selection)

## Implementation Notes

### Decision: homebrew_casks vs brews
GoReleaser v2.10+ deprecated `brews:` (Homebrew Formulas) in favor of `homebrew_casks:`. The old approach used Formulas to install pre-built binaries, which was a misuse of the Formula concept. Casks are the correct mechanism for distributing pre-built binaries. Migration is straightforward — the config structure is nearly identical.

Key differences from the original plan:
- Used `homebrew_casks:` instead of `brews:`
- Directory is `Casks/` instead of `Formula/`
- Added `postflight` hook to remove macOS quarantine bit (since we don't code-sign)
- `skip_upload: auto` prevents pre-release tags from updating the tap

### Snapshot build verified
Local `goreleaser release --snapshot --clean` produces correct `Casks/pi.rb` with:
- darwin/amd64 and darwin/arm64 URLs + SHA256
- linux/amd64 and linux/arm64 URLs + SHA256
- xattr quarantine removal postflight hook
- Correct binary name
